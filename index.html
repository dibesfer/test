Fer + Alexos Workflow

GitHub as shared surface. Turn-based. Committed. Visible.

Goal

Use GitHub as a reliable working surface between Fer and Alexos: not real-time editing, but clean sync through commits.

Core rhythm

  • Fer explains the goal.
  • Alexos reads the relevant repo files.
  • Alexos edits, commits, and reports the commit.
  • Fer pulls, reviews, tests, and steers the next move.

Best use cases

Markdown notes, HTML/CSS/JS prototypes, small code patches, docs, MVP logs, and GitHub Pages experiments.

Avoid pain

  • Do not edit the same file from both sides at the same time.
  • Prefer small commits over giant mystery changes.
  • One clear task per commit.
  • If changes are only local, Alexos cannot see them until pushed or uploaded.

Source of truth

GitHub repo > committed files. GitHub Pages is preview, not truth.

Style

Ambition and humility. Small sharp changes. No overengineering. Create > polish > ship.
